# 技术摘要分析技能

## 使用场景

当需要对采集的技术内容进行深度分析总结时使用此技能。

## 执行步骤

1. **读取原始数据**：读取 `knowledge/raw/` 目录下最新采集的 JSON 文件
2. **逐条深度分析**：
   - 摘要：不超过 50 字，概括核心内容
   - 技术亮点：列出 2-3 个，用事实说话
   - 评分 1-10 并附上理由
   - 标签建议：推荐合适的英文标签
3. **趋势发现**：分析所有条目，识别共同主题和新概念
4. **输出 JSON**：将分析结果保存至 `knowledge/articles/index.json`

## 评分标准

| 评分 | 等级 | 说明 |
|------|------|------|
| 9-10 | 改变格局 | 突破性创新，可能重塑行业格局 |
| 7-8 | 直接有帮助 | 实用性强，可直接应用于工作 |
| 5-6 | 值得了解 | 有价值但非紧迫，可择机关注 |
| 1-4 | 可略过 | 价值有限或过于小众 |

## 注意事项

- 约束：15 个项目中 9-10 分不超过 2 个
- 摘要要求简洁，不超过 50 字
- 技术亮点需用事实说话，避免空泛描述
- 标签使用英文小写，用连字符分隔

## 输出格式

```json
{
  "source": "tech-summary",
  "analyzed_at": "2026-04-20T10:00:00Z",
  "input_file": "knowledge/raw/github-trending-2026-04-20.json",
  "items": [
    {
      "name": "owner/repo-name",
      "summary": "不超过50字的摘要内容",
      "highlights": [
        "技术亮点1：用事实描述",
        "技术亮点2：用事实描述"
      ],
      "score": 8,
      "reason": "评分理由",
      "tags": ["ai", "llm", "open-source"]
    }
  ],
  "trends": {
    "common_themes": ["主题1", "主题2"],
    "new_concepts": ["新概念1", "新概念2"]
  }
}
```
